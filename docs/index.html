The Next Dungeon — Agent Teams

The current pipeline runs on sub-agents: a single orchestrator spawns QA, Security, and Tech Leader, and waits for each in turn. A new architecture is being mapped — Agent Teams, Claude's peer-autonomy model, where each reviewer operates as an independent agent, coordinating without a central dispatcher.

In an Agent Teams model, QA, Security, and Tech Leader would each hold their own context and communicate as peers — enabling longer, deeper reviews without a single session growing unbounded. Implementation in progress.
